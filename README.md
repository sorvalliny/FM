# FM Легенды

Интерактивная база данных вандеркидов всех версий Football Manager (2005—2024) и того, как сложилась их карьера в реальности — от GOAT до легендарных флопов.

🔗 **[Открыть базу](https://sorvalliny.github.io/FM/)**

## Что внутри

- **286 игроков** × 20 версий FM (14—15 на версию: ~3L + ~3T + ~3M + 5F)
- Рейтинг 1—10: от культового флопа (Аду, Самба, Бебе) до GOAT (Месси, Роналду, Мбаппе, Холанд)
- Фильтры по версии и категории рейтинга
- Поиск по имени, клубу, описанию

## Категории рейтинга

| Балл | Что значит |
|------|------------|
| 9—10 | Легенды мирового футбола |
| 7—8  | Топ-игроки, реализовали потенциал |
| 5—6  | Средняя карьера, не дотянули |
| 1—4  | Флопы и легендарные провалы |

## Распределение

- **61** легенд (9—10)
- **70** топ-игроков (7—8)
- **55** средних (5—6)
- **100** флопов (1—4) — главная коллекция

База сбалансирована по 20 версиям FM: минимум 3 игрока в каждой категории L/T/M + 5 флопов на версию.

## Самые точные предсказания FM

FM 2006 — 4 из 5 в категории 9—10: Рамос, Агуэро, Лам, Тевес.
FM 2017—2019 — золотое поколение: Мбаппе, Доннарумма, Винисиус, ТАА, Холанд, Фоден.

## Структура `data/players.json`

Массив записей, одна запись на строку «игрок × версия FM»:

```json
{
  "id": "lionel-messi-fm2005",
  "name": "Lionel Messi",
  "fm_version": "FM 2005",
  "club_at_release": "Barcelona B",
  "age_at_release": 17,
  "rating": 10,
  "verdict": "GOAT. 8 Золотых мячей...",
  "tags": [],
  "nationality": null,
  "position": null,
  "details": null,
  "sources": ["https://..."],
  "confidence": "verified"
}
```

| Поле | Обяз. | Что |
|------|:----:|-----|
| `id` | ✓ | slug, уникален. Формат: `имя-фамилия-fmYYYY` |
| `name` | ✓ | Имя игрока в оригинальном написании |
| `fm_version` | ✓ | `FM 2005` … `FM 2024` |
| `club_at_release` | ✓ | Клуб на момент релиза версии FM |
| `age_at_release` | ✓ | Возраст (integer) |
| `rating` | ✓ | 1—10 (1 = легендарный провал, 10 = GOAT) |
| `verdict` | ✓ | Короткий вердикт (1–2 предложения) |
| `tags` | — | Метки: `goat`, `flop`, `injuries`, `loan-king`, … |
| `nationality` | — | ISO-3 код (задел) |
| `position` | — | `GK`/`CB`/`LB`/`CM`/`AM`/`LW`/`ST`/… (задел) |
| `details` | — | Длинная биография (задел под карточку игрока) |
| `sources` | — | Массив URL с подтверждением вердикта |
| `confidence` | — | `verified` (есть источники) / `recall` (по памяти) / `legacy` (исходные 100) |

### Добавить игрока

1. Открой `data/players.json`
2. Добавь объект по схеме выше (обязательные поля — минимум)
3. Для вердикта — прикладывай `sources` (ссылки) и ставь `confidence: "verified"`; если без источников — `"recall"`
4. Проверь, что `id` уникален и JSON валиден: `python3 -m json.tool data/players.json > /dev/null`
5. Обнови цифры в README при необходимости

## Запуск локально

`file://` блокирует fetch, нужен локальный сервер:

```bash
cd fm-legends
python3 -m http.server 8080
# открой http://localhost:8080
```
