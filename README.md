# FM Легенды

Интерактивный рейтинг 100 самых хайповых вандеркидов всех версий Football Manager (2005—2024) и того, как сложилась их карьера в реальности. Стартовая точка для развития полноценной базы данных игроков FM.

🔗 **[Открыть рейтинг](https://sorvalliny.github.io/FM/)**

## Что внутри

- 20 версий FM × 5 игроков = 100
- Рейтинг 1—10: от культового флопа (Аду, Самба, Мастур) до GOAT (Месси, Роналду, Мбаппе, Холанд)
- Фильтры по версии и категории рейтинга
- Поиск по имени, клубу, описанию

## Категории рейтинга

| Балл | Что значит |
|------|------------|
| 9—10 | Легенды мирового футбола |
| 7—8  | Топ-игроки, реализовали потенциал |
| 5—6  | Средняя карьера, не дотянули |
| 1—4  | Флопы и легендарные провалы |

## Распределение

- **33** легенд (9—10)
- **37** топ-игроков (7—8)
- **17** средних (5—6)
- **13** флопов (1—4)

## Самые точные предсказания FM

FM 2006 — 4 из 5 в категории 9—10: Рамос, Агуэро, Лам, Тевес.
FM 2017—2019 — золотое поколение: Мбаппе, Доннарумма, Винисиус, ТАА, Холанд, Фоден.

## Самые культовые флопы

- **Фредди Аду** (FM 2005) — «новый Пеле», 14 клубов в 8 странах
- **Черно Самба** (CM/FM era) — 60+ виртуальных голов за сезон, 0 матчей в АПЛ
- **Керлон** (FM 2006) — «тюлений дриблинг», 51 матч за карьеру
- **Маркеда** (FM 2009) — легендарный гол «Вилле», после — пустота
- **Хашим Мастур** (FM 2014) — 0 матчей за Милан
- **Пьетро Пеллегри** (FM 2019) — €25M в 16, травмы съели карьеру

## Стек

- `index.html` — UI + рендер (vanilla JS, без зависимостей)
- `data/players.json` — база игроков (источник правды, легко править/расширять)
- Шрифты: Unbounded + Golos Text + JetBrains Mono с Google Fonts
- Тёмная тема, акценты violet/teal/amber

## Структура `data/players.json`

Массив записей, одна запись на строку «игрок × версия FM»:

```json
{
  "id": "lionel-messi-fm2005",
  "name": "Lionel Messi",
  "fm_version": "FM 2005",
  "club_at_release": "Barcelona B",
  "age_at_release": 17,
  "rating": 10,
  "verdict": "GOAT. 8 Золотых мячей...",
  "tags": [],
  "nationality": null,
  "position": null,
  "details": null
}
```

| Поле | Обяз. | Что |
|------|:----:|-----|
| `id` | ✓ | slug, уникален. Формат: `имя-фамилия-fmYYYY` |
| `name` | ✓ | Имя игрока в оригинальном написании |
| `fm_version` | ✓ | `FM 2005` … `FM 2024` |
| `club_at_release` | ✓ | Клуб на момент релиза версии FM |
| `age_at_release` | ✓ | Возраст (integer) |
| `rating` | ✓ | 1—10 (1 = легендарный провал, 10 = GOAT) |
| `verdict` | ✓ | Короткий вердикт (1–2 предложения) |
| `tags` | — | Метки: `goat`, `flop`, `injuries`, `loan-king`, … |
| `nationality` | — | ISO-3 код (задел) |
| `position` | — | `GK`/`CB`/`LB`/`CM`/`AM`/`LW`/`ST`/… (задел) |
| `details` | — | Длинная биография (задел под карточку игрока) |

### Добавить игрока

1. Открой `data/players.json`
2. Добавь объект по схеме выше (обязательные поля — минимум)
3. Проверь, что `id` уникален и JSON валиден: `python3 -m json.tool data/players.json > /dev/null`
4. Обнови цифры в README при необходимости
